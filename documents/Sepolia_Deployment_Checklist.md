# Sepolia Deployment Checklist

This document outlines the necessary steps to transition the KalaKrut platform from local development/mock data to the Sepolia Ethereum Testnet.

## 1. Current Goal
The immediate objective is to activate full Web3 functionality by bridging the frontend components to live smart contracts deployed on the Sepolia testnet. This will replace static/cached data with real-time on-chain state.

## 2. Contract Deployment Checklist
The following core contracts must be compiled and deployed to Sepolia in the specified order to maintain dependency integrity:

- [ ] **KalaKrutToken (ERC20)**: The governance and utility token.
- [ ] **KalaKrutNFT (ERC721)**: The standard for creative assets and fractionalization.
- [ ] **TimelockController**: Required for secure DAO governance execution.
- [ ] **KalaKrutGovernor**: The central DAO logic for proposals and voting.
- [ ] **Treasury**: The community vault for managing ETH and ERC20 assets.
- [ ] **ContractFactory**: For permissionless deployment of service agreements.
- [ ] **Escrow**: For secure project milestones and payments.
- [ ] **EventTicket**: For NFT-based event access and ticketing.
- [ ] **Fractionalizer**: For splitting NFT ownership into tradeable tokens.

## 3. Frontend Configuration
Once deployment is complete, the following source files must be updated:

### Contract Addresses
Update `src/data/contractAddresses.ts` with the new hex addresses generated by the Sepolia deployment:
```
typescript
export const contractAddresses = {
  KalaKrutToken: "0x...",
  Treasury: "0x...",
  // ... update all fields
};
```
### Wallet Connectivity
Ensure `src/contexts/WalletContext.tsx` (using Reown AppKit/Web3Modal) is configured with:
- **Chain**: `sepolia`
- **Project ID**: A valid WalletConnect `projectId`.

## 4. Functional Requirements
The Sepolia integration must support the following live actions:

- **DAO Governance**: Users must be able to create, view, and vote on proposals directly via the `DaoGovernorPage`.
- **Treasury Ledger**: The `TreasuryRealDataTable` must query the `FundsDeposited` and `FundsWithdrawn` events from the Sepolia Treasury contract.
- **Creative Studio**: Artists must be able to mint actual NFTs on Sepolia and view them in the Marketplace.
- **Automated Verification**: Profile verification should eventually be tied to on-chain credentials or NFT ownership.

## 5. Phase 3 Preview: Payments & Payouts
Following the successful Sepolia launch, the platform will move to Phase 3:
- Integration of Fiat-to-Crypto on-ramps.
- Real-world service payout automation via Escrow contracts.
- Hybrid accounting (Fiat + Crypto) finalized in the Treasury Ledger.
